Polynomials

inerpolation of functions mapping to a vector space

Download this project as a .zip file Download this project as a tar.gz file

This library should make it easy to construct and evaluate interpolation polynomials. for functions

state_type f( time_type t );

time_type is assumed to be one dimensional ( double, float ) and state_type can be any type supporting addition ad multiplication with time_type ( for example double or boost::numeric::ublas::vector).

Currently supported functions are

Feature syntax
construction polynomial<> P( {t0,t1,t2,...,tp}, {y0,y1,y2,...,yp} )
evaluation y = P( time )
differentiation y = P.deriv( time, order )
rootfinding t = P.findroot( rootfn, tlower, tupper )
differentiation polynomial<> deriv = P[order]
hermite interpolation polynomial<> P( {t0,t0,t0,t1}, {y0,dy0,ddy0,y1} )
integration P.integrate( t0, t1 )
addition/substraction polynomial<> sum = P1 + P2
adding points* P.add_point( tnew, ynew )
  • * limited support for adding derivatives ( that is tnew already exists in t_ )

plans

there are some missing features in the current implementation. Some major improvements developed right now are

  • automatic sorting of input, so that t_[0] <= t_[1] <= ... <= t_[p]
    • this requires add_point to insert the new point at an appropriate spot
      • tat in return requires to store all finite differences during construction
  • get an own class for polynomials in standard basis. This could be a derived class of polynomial supporting integration
  • set the identical function as default for rootfn if state_type == double. this way, a root of the polynomial is found which should be a common usecase.
